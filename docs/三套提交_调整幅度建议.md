## 一、2025 notebook 参考

| 来源 | 策略 | 调整方式 |
|------|------|----------|
| **季军 (3rd)** | 多队冠军主线 | 二元覆写：riskTeam 的比赛 Pred=1.0 或 0.0（到指定轮次） |
| **第7名 (goto-conversion)** | 单队风险 Drake | 二元覆写：Drake 前 2 轮 Pred=1.0/0.0 |
| **冠军 (1st)** | 渐变性调整 | ① pred<85% 时 +10% 相对提升 (x→1.1x)；② 部分场次 manual_overrides 到 0.95–0.98 |

**33.3% 法则**（goto-conversion）：在 Brier 下，假设「约 33% 胜率的一方赢」时，期望收益最大，因此最适合冒险的区间是模型给出 ≈25–40% 的对阵。

**关键结论**：获奖者变动幅度远比「小幅微调」大：

- **冠军 1st**：全局 `pred → pred×1.1`（0.6→0.66，+6% 绝对值），部分场次直接覆写到 **0.95–0.98**（原 0.6 左右时相当于 **+35%**）
- **季军 3rd**：冠军路径 **二元 0/1**，原 60%→100%（+40%），原 30%→100%（+70%）
- 若只用 +2~4% 微调，对 Brier 几乎无影响，难以拉开与其他提交的差距

## 四、最大化上限策略（激进押冠军）

若目标为**奖牌**，采用「三线激进押冠军、接受单线失败」的思路：

### 核心逻辑

- **前几轮（R1、R2）**：模型对冠军 vs 弱队 already 很高（0.85+），**少调或不调**，避免边际收益小、爆冷时惩罚大。
- **后期（S16、E8、F4、Final）**：冠军 vs 强队时模型常给 0.45–0.65，**集中激进上调**，把「赌注」放在这些场次。
- **单线失败可接受**：押错冠军的那条 submission 会明显掉分，但另外两条仍有机会冲奖牌；三条都押对则上限最高。

### 按轮次调整幅度（冠军路径内）

| 轮次 | 模型 pred 通常 | 建议操作 | 说明 |
|------|----------------|----------|------|
| R1 / R2 | 0.85–0.95 | **微调或不动**（+0~2%） | 已高，收益小 |
| S16 | 0.65–0.85 | **+5%~+10%** | 开始加大 |
| E8 | 0.50–0.75 | **+15%~+25%** 或设 0.82–0.90 | 核心赌注 |
| F4 / Final | 0.45–0.65 | **+20%~+35%** 或设 0.85–0.92 | 最大赌注 |

### 实现示意

```
对每条冠军主线 (Duke / Michigan / Arizona)：
  - R1、R2：pred 若 >0.88，不调；若 0.75–0.88，+1~2%
  - S16：pred += 0.08，clip(0.01, 0.98)
  - E8、F4、Final：pred = max(pred + 0.20, 0.85)，cap 0.92
```

即：前期保守、后期激进，把「赌」集中在冠军 vs 同档强队的场次。

### 策略取舍（为什么这样能拉高上限）

| 取舍 | 理由 |
|------|------|
| 前几轮少调 | 模型已给 0.85+，再调高边际收益小；若爆冷，高 pred 惩罚大。不在这消耗「赌注」。 |
| 后期大调 | E8/F4/Final 多为五五开或略优，模型常给 0.50–0.65。大幅上调到 0.85+，押对则 Brier 大赚；押错则该线本就会废，损失不额外放大。 |
| 单线失败可接受 | 三条 submission 分散在 Duke/Michigan/Arizona，只要有一条押对冠军，就有冲奖牌的可能；三条都保守则上限封顶，难拉开差距。 |
| 0.85–0.92 而非 1.0 | 季军用 0/1 二元，收益大但风险极高；用 0.88–0.92 在「押对时的收益」与「押错时的惩罚」之间取得较优平衡。 |

## 五、建议调整幅度（统一策略 / 不区分队伍，较保守版）

若采用**相同押注方式**但不想过于激进，三套 A/B/C 共用以下规则，仅冠军主线不同（Duke / Michigan / Arizona）：

### 1. 冠军 vs 弱队：高信心预测

| 对手强度 | 冠军胜率设置 | 说明 |
|----------|--------------|------|
| 对手 pairwise 排名 26–40 | **0.92–0.95** | 中弱队 |
| 对手 pairwise 排名 41+ | **0.95–0.97** | 弱队 |

注：Duke 历史爆冷率 71%，若更保守可对 Duke 用 0.90–0.93；Michigan 较稳定，可用 0.93–0.97。

### 2. 冠军 vs 中强/强队：渐进式调整

| 本方 pred 区间 | 调整幅度 |
|----------------|----------|
| 55–85% | +6% ~ +12% |
| 50–55% | +5% ~ +10% |
| 35–50%（被看衰） | +10% ~ +18% |
| >85% | 基本不动 |

### 3. 实现示意（统一逻辑）

```
对每条冠军主线 (Duke/Michigan/Arizona)：
  - 若冠军 vs 弱队：pred = 0.94（或按上表）
  - 若冠军 vs 中强/强队：pred += 调整幅度，clip(0.01, 0.98)
  - 其他比赛：不改
```

## 六、建议调整幅度（原分档逻辑，可弃用）

### 1. 通用原则（结合 GPT 与 Brier 特性）

- **0.50–0.70**：最适合作小修，幅度可稍大
- **0.70–0.85**：谨慎小修，避免拉得太高
- **0.85 以上**：除非有强依据，否则少动
- 不鼓励把 0.90 改成 0.97，爆冷时惩罚很大

---

### 2. A (Duke，保守)

| 对阵类型 | 调整对象 | 建议幅度 | 说明 |
|----------|----------|----------|------|
| **Duke vs 弱队** | Duke 胜率 | **直接设 0.92–0.97** | 高信心预测，弱队定义见下 |
| Duke 被看好（pred 55–85%） | Duke 胜率 | **+2% ~ +4%** | 小幅上修，不激进 |
| Duke 略被看好（pred 50–55%） | Duke 胜率 | **+1% ~ +3%** | 更保守 |
| Duke 被看衰（pred 35–50%，如 vs Michigan） | Duke 胜率 | **+3% ~ +6%** | 可略多给 Duke，但不猛拉 |
| Duke 极度被看好（pred >85%） | 维持不变 | 0% | 避免放大爆冷损失 |

**弱队定义**（用于「Duke vs 弱队」高信心规则）：

- 用 pairwise 排名划分：对手排名 **26+**（中弱/弱队）或 **40+**（更弱），可视为弱队
- 建议：对手排名 26–40 时 Duke 胜率设 **0.92–0.95**；对手排名 41+ 时设 **0.95–0.97**
- 参考冠军 1st 的 manual_overrides（0.95–0.98），对强弱悬殊的对阵给出明确结论

整体：Duke 对弱队给高信心；对中强/强队小步上修，主要针对 0.5–0.85 区间。

### 4. C (Arizona，强调上限)

更偏进攻，尤其在「Arizona 略被看衰」的对阵上多做文章（33.3% 法则适用区间）：

| 对阵类型 | 调整对象 | 建议幅度 | 说明 |
|----------|----------|----------|------|
| Arizona 被看好（pred 55–85%） | Arizona 胜率 | **+3% ~ +6%** | 比 A/B 更积极 |
| Arizona 略被看好（pred 50–55%） | Arizona 胜率 | **+2% ~ +5%** |
| Arizona 被看衰（pred 25–45%，vs Duke/Michigan 等） | Arizona 胜率 | **+8% ~ +14%** | 核心风险区，可更激进 |
| pred >85% | 维持不变 | 0% |

重点：当模型给 Arizona 约 30–40% 时，可以上修到约 40–50%，这是 33.3% 法则附近区间。

## 四、实现建议

1. **用相对还是绝对调整**
   - 绝对：`new_pred = pred + 0.04`（简单，易失控）
   - 相对：`new_pred = pred + 0.1 * pred`（冠军 1st 类似，对高 pred 影响更大）
   - 建议：优先用**绝对调整**，且加 `clip(0.01, 0.99)`，避免越界

2. **轮次分级（可选）**
   - 首轮、次轮：偏保守（+1% ~ +3%）
   - 16 强、8 强：可略加大（+2% ~ +5%）
   - 4 强、决赛：可最大（+4% ~ +8%，C 方案可到 +14%）

3. **仅改「冠军主线相关」的比赛**
   - 不改无关对阵，避免对 Brier 的全局破坏
   - 季军做法：只对 riskTeam 参与的比赛覆写
