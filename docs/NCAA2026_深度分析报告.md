# NCAA March Madness 2026 深度分析报告与初步方案

## 摘要

本报告对 Kaggle NCAA March Madness 竞赛进行深度剖析，梳理竞赛机制、评分体系，分析历年获奖方案中的三大核心方法（Raddar 框架、goto_conversion、手动覆盖策略），并给出面向 2026 赛季的初步技术方案。

---

## 一、竞赛概况

### 1.1 基本信息

- **竞赛官网**: [march-machine-learning-mania-2026](https://www.kaggle.com/competitions/march-machine-learning-mania-2026)
- **任务本质**: 预测 NCAA 锦标赛（男女篮）每场**可能对阵**中，**低 TeamID 球队**的获胜概率
- **特点**: 更偏「结果导向」——不仅考验机器学习模型，还需结合博彩/专家信息与策略性押注

### 1.2 提交格式

根据 `data_explained` 与 `SampleSubmissionStage1.csv`：

| 字段 | 格式 | 说明 |
|------|------|------|
| ID | SSSS_XXXX_YYYY | 赛季(4位) + 小TeamID(4位) + 大TeamID(4位) |
| Pred | [0, 1] | 第一个队（XXXX）的获胜概率 |

- **Stage1**: 2022–2025 年所有对阵，用于本地验证
- **Stage2**: 当季所有对阵，正式提交用

### 1.3 评分指标：Brier Score

使用 **Brier Score**（预测概率与 0/1 真实结果的均方误差）：

```
Brier = mean((Pred - actual)^2)
```

其中 `actual = 1` 表示该队赢，`actual = 0` 表示该队输。**分数越低越好**。各 notebook 均使用 `sklearn.metrics.brier_score_loss` 评估。

### 1.4 Brier 与「押注策略」的数学关系

`updated-goto-conversion-winning-solution.ipynb` 中给出的数学证明：

期望收益可表示为 `f(p) = p(1-p)^2`，其中：
- `p` 为某队获胜的真实概率
- `(1-p)^2` 为押该队赢且猜对时的 Brier 收益

求导得 `argmax_p f(p) = 1/3`。

**结论**: 在 Brier 评分下，对胜率约 **33.3%** 的队伍「押赢」（Pred=1），若猜对则期望收益最大。这驱动了后文「风险队连胜押注」策略——选择约 33% 胜率的冷门队，在其可能晋级的前几轮中强制 Pred=1。

## 二、三大核心方法

### 2.1 Raddar 框架（vilnius-ncaa.ipynb）

原作者 **raddar**，多届金/银/铜牌得主公开引用该 notebook 作为基础框架。

#### 数据流

```
RegularSeasonDetailedResults + MNCAATourneyDetailedResults
        ↓ prepare_data (W/L 对调翻倍)
regular_data, tourney_data (每场 2 行，T1/T2 对称)
        ↓ 丢弃本场 box score，保留 Season, T1_TeamID, T2_TeamID, PointDiff, win, men_women
        ↓ merge seeds, 常规赛场均, Elo, GLM quality
tourney_data with features
        ↓ XGBoost (reg:squarederror) + UnivariateSpline 校准
Pred (胜率)
```

#### 特征工程

| 类别 | 特征 | 来源 |
|------|------|------|
| 基础 | men_women, T1_seed, T2_seed, Seed_diff | seeds |
| 常规赛 | T1_avg_Score, T1_avg_FGA, T1_avg_Blk, T1_avg_PF 等 | regular_data groupby |
| 防守侧 | T1_avg_opponent_*, T2_avg_opponent_* | 对手在面对该队时的场均 |
| 评级 | T1_elo, T2_elo, elo_diff | 按比赛顺序更新的 Elo |
| 质量 | T1_quality, T2_quality | GLM: PointDiff~-1+T1_TeamID+T2_TeamID |

**Elo 参数**: `base_elo=1000`, `elo_width=400`, `k_factor=100`  
**GLM**: 每个赛季、男女分别拟合，系数即队伍强度（quality）

#### 模型与校准

- **XGBoost**: `reg:squarederror`，预测 PointDiff；leave-one-season-out 交叉验证
- **UnivariateSpline**: 将分差预测映射为 [0,1] 胜率；`np.clip(0.01, 0.99)` 防止极端值

### 2.2 goto_conversion

- **来源**: [github.com/gotoConversion/goto_conversion](https://github.com/gotoConversion/goto_conversion)
- **作用**: 将博彩赔率转为概率，缓解 favourite-longshot bias

#### 核心算法

对逆赔率 `1/odds` 做统一 SE 步长调整：

```python
listOfProbabilities = 1.0 / listOfOdds
listOfSe = sqrt((p - p^2) / p)
step = (sum(p) - 1) / sum(Se)
outputProbabilities = p - (Se * step)
```

即：逆赔率按各自标准误差比例「回退」，使总概率和为 1。

#### 产出

- `mensProbabilitiesTable.csv`, `womensProbabilitiesTable.csv`
- 列: `rd1_win`, `rd2_win`, ..., `rd6_win`（晋级各轮概率）
- 需要 538 等外部赔率数据；本仓库有 2024/2025 概率表

---

### 2.3 手动覆盖策略（Manual Override）

#### 形式一：固定对阵覆盖

对少数「强弱分明」的早期对阵，直接设高 Pred：

```python
manual_overrides = {
    '2025_1124_1280': 0.98185013,
    '2025_1140_1433': 0.980304479,
    ...
}
```

#### 形式二：风险队连胜押注（set_optimalStrategy）

根据 bracket 推断对阵出现的轮次，对指定「风险队」在其可能晋级的前若干轮内强制 Pred=1 或 0。

**核心函数**:
- `get_roundOfMatch(team1, team2, seeds_df)`: 根据种子与 bracket 结构推断轮次
- `get_tourneyFlag`: 判断两队是否均在锦标赛内
- `set_optimalStrategy`: 若 `submission_round <= riskTeamToWinRound` 且风险队为 team1，则 Pred=1；若为 team2，则 Pred=0

**示例**:
```python
riskTeam = 1179  # Drake
riskTeamToWinRound = 2  # 押其赢前 2 轮
set_optimalStrategy(submission_df, mensSeeds, womensSeeds, riskTeam, riskTeamToWinRound)
```

## 四、初步技术方案

### 4.1 总体架构

```
[Raddar XGB+spline]  ─┐
                      ├─→ [Blend / 加权] ─→ [set_optimalStrategy] ─→ [manual_overrides] ─→ submission.csv
[goto_conversion 概率] ─┘
```

### 4.2 分阶段实施

**Phase 1 - 复现与验证**
- 以 `vilnius-ncaa.ipynb` 为主干，跑通数据、特征、模型、Brier 评估
- 适配 2026 数据路径（`march-machine-learning-mania-2026`）
- 复现 goto_conversion 流程：若有 538 2026 赔率则生成新概率表；否则先用 2025 表做验证

**Phase 2 - 融合与调参**
- 将 goto 概率作为**特征**并入 Raddar（如 T1_538_prob, T2_538_prob），或做预测级 blend
- 调 XGB 超参、spline 校准、blend 权重，用 leave-one-season-out Brier 选优

**Phase 3 - 覆盖策略**
- 实现 `set_optimalStrategy`（支持多队、多轮）
- Selection Sunday 后，结合 bracket 与专家判断确定风险队与轮次
- 可选：对明显强弱对阵做少量 manual_overrides（参考 1st 的 6 场）

### 4.3 可交付物

- 本报告：`docs/NCAA2026_深度分析报告.md`
- 模块化代码：`src/` 下可复用模块

  - `src/raddar_utils.py`: prepare_data, extract_seed_number
